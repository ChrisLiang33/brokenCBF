Two-level hierarchy

The system has two distinct policies stacked. Don't confuse them.

LayerInputsOutputStatus
Teacher (we train this) env class (priv), grid, height, tracking_err CBF params (α, …) active training (~50 Hz)
Locomotion controller u_safe + full proprio (joints, IMU, body vel) joint torques pre-trained, frozen

The teacher's role is parameter selection — "given the situation, how aggressive should the safety filter be?" It doesn't need joint angles for that. Joint-level state belongs to the locomotion layer, which is a solved problem (Unitree Go2 flat velocity tracking) treated as a frozen black box.

When you see tracking_err in the teacher's priv input, that's the locomotion layer "reporting up" how well it tracked the last command. The teacher uses it as a coarse "am I keeping up?" signal — useful for setting α when the lower controller is struggling (slippery floor, heavy load) without needing the full proprio stack.

The adversarial planner uses global ground-truth obstacle positions to choose where to drive. The CBF QP and the policy are both FOV-limited (3.2 m). So the "evil driver" knows about everything; the car's safety system only sees what its LiDAR sees. Matches the AI-car narrative: planner picks dangerous commands, robot has to defend with limited perception.

Only the priv encoder changes between teacher and student. Grid CNN, policy MLP, and CBF QP transfer unchanged. The sim-to-real gap is isolated to one small regressor whose job is "infer env class from (state, action) history."

Why this architecture

v3.0a-d failure mode: monolithic encoder. Grid CNN (~530k params, per-step signal) dominated PPO gradient credit, starved dyn path (~1.3k params, per-episode signal). Z encoded geometry, ignored env class.

v3.0e/f failure mode: even with pretrained encoder, policy saturated at α=5 because reward stack + aux loss + velocity reward all pushed α up, only sparse fall penalty pulled down. Aux loss self-silenced when slack stayed positive.

v3.2 design response:

  • Split bottlenecks — z_priv and z_grid don't share parameters. No gradient-credit competition. Each path trains on its own signal.
  • cbf_state out of policy — no shortcut. α must be inferred from env class (z_priv) + raw perception (z_grid), not derived CBF features.
  • AUX_COEF = 0 — removes the self-silencing aux loss. Saturation pressure is gone; signal density comes from data instead.
  • 45% adversarial + FOV-gated CBF — makes friction/mass action-relevant. Policy must encode env class to set α correctly. The data carries the gradient that aux loss was failing to provide.
