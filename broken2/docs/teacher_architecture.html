3. Why φ still doesn't use the grid features (within episode)

The architecture could drive φ from grid features — z_grid feeds π_teacher which outputs all 5 CBF params including φ. But the per-episode φ lock in _cbf_filter blocks it:

z_grid (64-D)
varies each step → π_teacher → policy φ_t
(varies)
↓
PER-EPISODE LOCK
captures φ at step 0, ignores all later φ_t
↓
CBF actual φ
= φ_step_0 (constant)

So even though z_grid (and hence the policy's φ output) is changing every step as the robot moves through the scene, the locked CBF discards those changes. The grid CNN's spatial features never get to influence φ within an episode.

4. What V4 (windowed lock) changes

z_grid

↓
WINDOWED LOCK @ 5s
refreshes every 250 steps within episode
↓
CBF actual φ
= step-0 φ (250 steps), then refresh, then 250 steps of new φ, ...

Robot traverses open → corridor → open within one episode. z_grid sees this. Policy's φ_t output reflects it. With windowed lock, the CBF gets ~4 different φ values per 20s episode — enough to switch from "open: low φ" to "corridor: higher φ" mid-episode.

5. On switching the teacher to LiDAR (your suggested fix)

You asked if we should switch the teacher off GT positions and onto LiDAR. The honest answer:

  • RMA's design intent: teacher uses privileged info (because it's available in sim and gives faster/cleaner training), student distills that knowledge using LiDAR (because that's what's available at deploy). If teacher uses LiDAR too, we lose the privileged advantage and the teacher trains slower / worse.
  • What we'd gain: smaller teacher-student gap (since both see the same input distribution). Helpful for distillation.
  • What we'd lose: teacher convergence speed + cleanliness of the latent z_priv. Teacher would have to do perception inference too.
  • Practical recommendation: keep teacher on GT-derived grid (current). The CNN is already there. The grid features are already feeding the policy. The bottleneck for φ adaptation isn't perception — it's the lock.

6. The actual fix path

  1. V4 = unlock φ to use the grid features it already has access to. Windowed lock @ 5s. Single config change from V3.
  2. If V4 shows within-episode φ tracking obstacle proximity (Pearson(φ_t, h_t) strongly negative within episode), the grid → φ pipeline is working. Step 2 is on track.
  3. If V4 doesn't show within-episode adaptation despite the unlock, the bottleneck is somewhere else (reward landscape, gradient noise, or grid encoder capacity).
